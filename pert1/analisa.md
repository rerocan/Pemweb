# **Analisis 5W1H website**

#### **1. WHAT (Apa?)**

- **Apa itu website?**
Website (situs web) adalah kumpulan halaman digital yang saling terhubung, berisi teks, gambar, video, atau konten lainnya, yang dapat diakses melalui internet menggunakan browser (seperti Chrome, Firefox, atau Safari). Setiap website memiliki alamat unik yang disebut URL (Contoh: www.namawebsite.com).

- **Apa saja Komponen yang ada di website?**
    *   Domain Name
Nama unik yang menjadi alamat website (Contoh: google.com, instagram.com).
Analogi: Seperti "alamat rumah" di dunia digital.

    * Hosting
    Tempat penyimpanan data website (file, gambar, kode program) di server yang terhubung ke internet.
    Analogi: Seperti "tanah" tempat rumah (website) dibangun.

    * Halaman Web (Web Pages)
        Dokumen individual yang ditulis dalam bahasa pemrograman (HTML, CSS, JavaScript) dan ditampilkan di browser.
        Contoh: Halaman beranda (homepage), halaman kontak, atau halaman produk.
- **Apa tujuan website?**
Contoh: Menjual produk (e-commerce), menyediakan informasi edukasi, blog personal, atau platform layanan jasa.
- **Apa fitur utama website?**
  Contoh: Katalog produk, formulir kontak, sistem pembayaran online, blog artikel, atau integrasi media sosial.


#### **2. WHO (Siapa?)**

- **Siapa target pengguna website?**Contoh: Remaja, profesional, ibu rumah tangga, atau pelaku bisnis.
- **Siapa pembuat/pengelola website?**
  Contoh: Tim internal perusahaan, freelancer, atau agensi digital.

#### **3. WHERE (Di mana?)**

- **Di mana website dihosting?**Contoh: Hosting lokal (Contoh: Niagahoster) atau server internasional (Contoh: AWS, Cloudflare).
- **Di mana pengguna mengakses website?**
  Contoh: Mayoritas dari Indonesia, perkotaan, atau global.

#### **4. WHEN (Kapan?)**

- **Kapan website diluncurkan?**Contoh: 2020, 2023, atau masih dalam tahap pengembangan.
- **Kapan update konten terakhir dilakukan?**
  Contoh: Blog diupdate mingguan, produk baru ditambahkan bulanan.

#### **5. WHY (Mengapa?)**

- **Mengapa website ini dibuat?**
  Contoh: Memperluas pasar, meningkatkan penjualan, membangun personal branding, atau menyediakan layanan online.

#### **6. HOW (Bagaimana?)**

- **Bagaimana website diakses?**Contoh: Responsif di desktop/mobile, kecepatan loading 3 detik, menggunakan HTTPS.
- **Bagaimana cara pengguna berinteraksi?**Contoh: Chatbot, formulir pendaftaran, atau fitur ulasan produk.
- **Bagaimana teknologi yang digunakan?**
  Contoh: WordPress, React.js, PHP, atau CMS custom.

# **Analisis S.W.O.T website**

#### **Strengths (Kekuatan)**

- **Konten Berkualitas**: Artikel/blog informatif atau produk unik yang sulit ditemukan di kompetitor.
- **User Experience (UX) Baik**: Navigasi intuitif, desain menarik, dan responsif di semua perangkat.
- **Keamanan**: SSL terpasang, sistem pembayaran terenkripsi, proteksi dari serangan DDoS.

#### **Weaknesses (Kelemahan)**

- **Loading Lambat**: Performa website di bawah standar karena gambar tidak teroptimasi.
- **SEO Buruk**: Konten tidak teroptimasi untuk mesin pencari, ranking rendah di Google.
- **Kurang Interaktif**: Tidak ada fitur live chat atau respon pelanggan yang lambat.

#### **Opportunities (Peluang)**

- **Potensi Pasar**: Permintaan produk/jasa meningkat di niche tertentu (misal: sustainable fashion).
- **Teknologi Terbaru**: Integrasi AI untuk rekomendasi produk atau chatbot.
- **Ekspansi Audiens**: Menargetkan pasar internasional dengan fitur multi-bahasa.

#### **Threats (Ancaman)**

- **Kompetisi Ketat**: Banyak kompetitor dengan harga lebih murah atau fitur lebih lengkap.
- **Perubahan Kebijakan**: Contoh: perubahan algoritma Google yang menurunkan traffic.
- **Risiko Keamanan**: Serangan hacker atau kebocoran data pengguna.

# **Analisis 5W1H Docker**

#### **1. WHAT (Apa?)**  
- **Apa itu Docker?**  
  Docker adalah platform **containerization** untuk mengemas aplikasi dan dependensinya ke dalam kontainer yang ringan, portabel, dan konsisten di berbagai lingkungan.  

- **Apa saja komponen yang ada di Docker?**  
  - **Container**: Lingkungan terisolasi untuk menjalankan aplikasi.  
    *Analogi: Seperti "kardus" yang berisi semua kebutuhan aplikasi.*  
  - **Docker Image**: Template read-only untuk membuat kontainer.  
  - **Dockerfile**: Skrip untuk membangun image secara otomatis.  
  - **Docker Hub**: Repositori publik untuk berbagi image (contoh: image MySQL, Nginx).  

- **Apa tujuan Docker?**  
  Contoh: Mempermudah deployment aplikasi, memastikan konsistensi antar lingkungan (dev, staging, production).  

- **Apa fitur utama Docker?**  
  Contoh: Isolasi aplikasi, portabilitas, integrasi dengan CI/CD, skalabilitas menggunakan Kubernetes.  

---

#### **2. WHO (Siapa?)**  
- **Siapa target pengguna Docker?**  
  Contoh: Developer, tim DevOps, perusahaan IT, atau cloud engineer.  
- **Siapa pembuat/pengelola Docker?**  
  Contoh: Docker, Inc. (perusahaan pengembang), komunitas open-source, atau tim internal perusahaan.  

---

#### **3. WHERE (Di mana?)**  
- **Di mana Docker dihosting?**  
  Contoh: Server cloud (AWS, Google Cloud), server on-premise, atau laptop developer.  
- **Di mana Docker digunakan?**  
  Contoh: Lingkungan microservices, pipeline CI/CD, atau aplikasi berbasis cloud.  

---

#### **4. WHEN (Kapan?)**  
- **Kapan Docker diluncurkan?**  
  Contoh: Rilis pertama tahun 2013, dengan versi stabil terus diperbarui (contoh: Docker Desktop 4.25).  
- **Kapan Docker digunakan?**  
  Contoh: Saat membangun aplikasi multi-layanan, migrasi ke cloud, atau memperbaiki konsistensi lingkungan kerja.  

---

#### **5. WHY (Mengapa?)**  
- **Mengapa Docker penting?**  
  Contoh: Mengurangi konflik dependensi, mempercepat proses development, dan memudahkan skalabilitas aplikasi.  

---

#### **6. HOW (Bagaimana?)**  
- **Bagaimana Docker bekerja?**  
  Contoh: Mengisolasi aplikasi menggunakan kernel OS host, menjalankan kontainer dari image yang telah dibuat.  
- **Bagaimana cara pengguna berinteraksi?**  
  Contoh: Perintah CLI (`docker run`, `docker build`) atau antarmuka GUI seperti Docker Desktop.  
- **Bagaimana teknologi yang digunakan?**  
  Contoh: Container runtime (containerd), orchestration (Kubernetes), atau registri (Docker Hub).  

---

# **Analisis S.W.O.T Docker**

#### **Strengths (Kekuatan)**  
- **Portabilitas**: Kontainer bisa dijalankan di semua OS yang mendukung Docker.  
- **Efisiensi Resource**: Lebih ringan dibanding virtual machine (VM).  
- **Komunitas Besar**: Dukungan dari Docker Hub dan dokumentasi lengkap.  

#### **Weaknesses (Kelemahan)**  
- **Kurang Cocok untuk GUI**: Fokus pada aplikasi CLI/backend.  
- **Kompleksitas Jaringan**: Konfigurasi jaringan antar kontainer rumit untuk pemula.  
- **Keamanan**: Risiko kerentanan jika image tidak di-update.  

#### **Opportunities (Peluang)**  
- **Adopsi Cloud-Native**: Tren microservices dan serverless computing.  
- **Integrasi dengan AI/ML**: Kontainer untuk deployment model machine learning.  
- **Ekosistem Kubernetes**: Docker sebagai fondasi orchestration.  

#### **Threats (Ancaman)**  
- **Kompetisi**: Teknologi alternatif seperti Podman atau containerd.  
- **Perubahan Regulasi Cloud**: Kebijakan vendor cloud yang membatasi penggunaan kontainer.  
- **Overhead Manajemen**: Kompleksitas saat aplikasi semakin besar.

# **Analisis 5W1H HTML**

#### **1. WHAT (Apa?)**  
- **Apa itu HTML?**  
  HTML (HyperText Markup Language) adalah bahasa markup untuk membuat struktur halaman web.  

- **Apa saja komponen yang ada di HTML?**  
  - **Tag**: Elemen sintaks untuk menandai konten (contoh: `<h1>`, `<div>`).  
    *Analogi: Seperti "kerangka bangunan" untuk web.*  
  - **Atribut**: Informasi tambahan dalam tag (contoh: `href` pada `<a>`).  
  - **Elemen**: Kombinasi tag, konten, dan atribut.  

- **Apa tujuan HTML?**  
  Contoh: Membangun struktur dasar website, mengatur tata letak teks/gambar.  

- **Apa fitur utama HTML?**  
  Contoh: Semantic tags (HTML5), embed multimedia, formulir interaktif.  

---

#### **2. WHO (Siapa?)**  
- **Siapa target pengguna HTML?**  
  Contoh: Web developer, desainer UI/UX, atau pemula yang belajar coding.  
- **Siapa pembuat/pengelola HTML?**  
  Contoh: World Wide Web Consortium (W3C), komunitas web developer.  

---

#### **3. WHERE (Di mana?)**  
- **Di mana HTML dihosting?**  
  Contoh: File HTML disimpan di server hosting atau platform static site (Netlify).  
- **Di mana HTML digunakan?**  
  Contoh: Browser web, aplikasi hybrid (React Native), atau email HTML.  

---

#### **4. WHEN (Kapan?)**  
- **Kapan HTML diluncurkan?**  
  Contoh: Versi pertama (1991), HTML5 rilis tahun 2014.  
- **Kapan HTML digunakan?**  
  Contoh: Setiap kali membuat halaman web, baik statis atau dinamis.  

---

#### **5. WHY (Mengapa?)**  
- **Mengapa HTML penting?**  
  Contoh: Standar universal untuk web, kompatibel dengan CSS/JavaScript.  

---

#### **6. HOW (Bagaimana?)**  
- **Bagaimana HTML bekerja?**  
  Contoh: Browser membaca file HTML dan merender konten sesuai tag.  
- **Bagaimana cara pengguna berinteraksi?**  
  Contoh: Klik tautan, isi formulir, atau navigasi menu.  
- **Bagaimana teknologi yang digunakan?**  
  Contoh: Dikombinasikan dengan CSS (styling) dan JavaScript (interaktivitas).  

---

# **Analisis S.W.O.T HTML**

#### **Strengths (Kekuatan)**  
- **Standar Universal**: Dukung semua browser modern.  
- **Mudah Dipelajari**: Sintaks sederhana untuk pemula.  
- **Fleksibel**: Bisa diintegrasikan dengan framework (React, Angular).  

#### **Weaknesses (Kelemahan)**  
- **Statis**: Tidak bisa membuat logika tanpa JavaScript.  
- **Bergantung pada Teknologi Lain**: Harus dipadukan dengan CSS/JS untuk hasil maksimal.  
- **Inkonsistensi Render**: Perbedaan tampilan di browser tertentu.  

#### **Opportunities (Peluang)**  
- **Web 3.0**: Adaptasi ke teknologi decentralisasi (blockchain).  
- **PWA (Progressive Web Apps)**: HTML sebagai dasar aplikasi offline.  
- **IoT**: Integrasi dengan perangkat pintar via web.  

#### **Threats (Ancaman)**  
- **No-Code Tools**: Platform seperti Wix yang mengurangi kebutuhan coding manual.  
- **Native Apps**: Persaingan dengan aplikasi mobile (Android/iOS).  
- **Perubahan Standar**: Update W3C yang memerlukan adaptasi terus-menerus.
